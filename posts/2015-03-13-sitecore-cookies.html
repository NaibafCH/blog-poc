Das Sitecore CMS verwendet diverse Cookies. Eine genaue Auflistung der verwendeten Cookies gibt es allerdings nicht.
Eine Auflistung (aktuelle Liste zu Sitecore 8) mit den gängisten findet sich hier:

Cookie Beschreibung
.ASPXAUTH ASP.NET Cookie, das bestimmt ob der User authentifiziert ist oder nicht. Der Wert ist mit dem MachineKey encrypted.
ASP.NET_SessionId ASP.NET Cookie, das automatisch beim Aufruf erstellt wird. Beinhaltet den Sessionstatus.
__CSRFCOOKIE Security Cookie zum verhindern von Anfragefälschungen. CSRF steht für Cross-Site-Request-Forgery.
__RequestVerificationToken Security Cookie zum verhindern von Anfragefälschungen.
__CSRFCOOKIE Security Cookie zum verhindern von Anfragefälschungen. CSRF steht für Cross-Site-Request-Forgery.
Sitecore DMS
SC_ANALYTICS_GLOBAL_COOKIE Das Sitecore Analytics Cookie misst den Traffic der Website und identifiziert den Besucher über die Session hinweg. Das Cookie wird also mit der Session nicht gelöscht, sondern wird erst nach einem Jahr ersetzt.
SC_ANALYTICS_SESSION_COOKIE Das Sitecore Analytics Cookie verfolgt den Besucher während seiner Session. Beim schliessen und erneuten öffnen des Browsers startet eine neue Besucher-Session mit neuer Cookie ID.
Sites (Cookies, die pro Site angelegt werden)
{site}#lang (website#lang) Die aktuelle Spracheinstellung. Man beachte, dieses Cookie wird für jede in Sitecore eingerichtete Site angelegt. Das Cookie greift erst wenn in der Url weder ein Sprache im Pfad (/en) noch QueryString (sc_lang=en) angegeben wurde.
{site}#sc_mode (website#sc_mode) Entspricht derselben Funktion wie als QueryString Parameter. Gibt also an in welchem Modus man sich befindet. (Mögliche Werte: edit, preview, normal)
{site}#sc_debug (website#sc_debug) Entspricht derselben Funktion wie als QueryString Parameter. Gibt über die Werte 1 und 0 an, ob man sich im Debug Modus von Sitecore befindet.
{site}#sc_prof (website#sc_prof) Entspricht derselben Funktion wie als QueryString Parameter. Gleichzusetzen mit Debug Profile Button "Activate".
{site}#sc_rb (website#sc_rb) Entspricht derselben Funktion wie als QueryString Parameter. Gleichzusetzen mit Debug Checkbox "Rendering Borders".
{site}#sc_ri (website#sc_ri) Entspricht derselben Funktion wie als QueryString Parameter. Gleichzusetzen mit Debug Checkbox "Rendering Information".
{site}#sc_trace (website#sc_trace) Entspricht derselben Funktion wie als QueryString Parameter. Gleichzusetzen mit Debug Trace Button "Activate".
{site}#sc_date (website#sc_date) Das Selektierte Datum im Page Editor für die Preview des Inhaltes zu einer bestimmten Zeit.
Path Analyzer
scPathAnalyzer_treeDefinition Path Analyzer Map Selektion. Damit wird die letzte Auswahl gespeichert.
scPathAnalyzer_fromDate Path Analyzer Filter Range Startwert. Damit wird die letzte Auswahl gespeichert.
scPathAnalyzer_toDate Path Analyzer Filter Range Endwert. Damit wird die letzte Auswahl gespeichert.
Experience Analytics
scExperienceAnalytics_FromDate Experience Analytics Filter Range Startwert.
scExperienceAnalytics_ToDate Experience Analytics Filter Range Endwert.
scExperienceAnalytics_Subsite Experience Analytics Filter. Die Auswertungen können auf gewisse Sites gefiltert werden.
Experience Editor
sc_last_page_mode_command Speichert das letzte abgesetzte Kommando aus dem Page Editor.
sitecore_webedit_activestrip Speichert, welcher Ribbon Reiter zuletzt selektiert war, um bei erneutem Aufruf den Tab wieder zu öffnen. (Beispielwerte: AdvancedStrip_ribbon_tab, ViewStrip_ribbon_tab)
Explore View
sc_expview Gibt an ob die Explore View geöffnet ist. Wert 1 für geöffnet.
left Bestimmt, ob die linke Akkordeonansicht geöffnet ist oder nicht. (false = geöffnet)
right Bestimmt, ob die rechte Akkordeonansicht geöffnet ist oder nicht. (false = geöffnet)
accordions-editor Speichert welche Akkordeon Lamelle in der linken Akkordeonansicht geöffnet ist. (Speichert ID)
accordions-viewer Speichert welche Akkordeon Lamelle in der rechten Akkordeonansicht geöffnet ist. (Speichert ID)
Andere
sitecore_userticket Cookie wird an Autoren vergeben, fürs CMS Backend.
sc_fv Flash Version
sc_pview_shuser Wird benutzt beim öffnen der Preview und speichert Preview und Debugging Optionen.
scContentEditorFoldersWidth Dieses Cookie speichert die Breite der Tree-Navigation des Content Editors. Das heisst: Ändert sich die Breite der Navigation wird damit auch der Cookie Wert auf die entsprechende Pixel Breite gesetzt.

Weitere Cookies

Fehlende Cookies dürfen gerne über die Kommentarfunktion ergänzt werden.
